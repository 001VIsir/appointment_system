# 一分钟版项目介绍（面试开场）

大家好，我做的是一个 ToC 预约平台。核心是让商户创建预约任务，用户通过短期签名链接直接预约，管理员负责全局统计与监管。

整体采用前后端分离：前端 React，后端 Spring Boot REST，MySQL 存业务数据，Redis 做 Session 和限流。

我重点负责预约任务体系、签名链接访问与并发一致性。预约下单使用数据库锁，确保高并发不超卖；签名链接是 `taskId + 过期时间 + HMAC`，有效期短，避免长期 token 泄露风险。

系统还提供统计接口与调用监控，方便运维与业务分析。后续可扩展到消息队列削峰、API 版本控制和 SDK 接入。
